0. UI ภาพรวม
ระบบถูกออกแบบมาเพื่อรองรับกระบวนการขอลบบัญชีตามหลักกฎหมายคุ้มครองข้อมูลส่วนบุคคล (PDPA) โดยแบ่งเป็น 2 ส่วนหลัก:

  - User Interface: หน้าจอสำหรับผู้ใช้ทั่วไปเพื่อส่งคำขอและระบุเหตุผล
  [ภาพรวมหน้าจอผู้ใช้](./images/ui-user-profile.png)

  - Admin Dashboard: หน้าจอสำหรับผู้ดูแลระบบในการตรวจสอบ อนุมัติ หรือปฏิเสธคำขอ
  [ภาพรวมหน้าจอแอดมิน](./images/ui-admin-all-requests.png)

1. สำหรับผู้ใช้ทั่วไป (General User)
1.1 การเข้าถึงฟีเจอร์

- ผู้ใช้สามารถเข้าถึงฟีเจอร์ได้โดยไปที่เมนู "โปรไฟล์" (Profile) จากนั้นมองหาปุ่มสีแดงที่เขียนว่า "ลบบัญชี" ที่มุมขวาล่างของหน้าจอ

1.2 ขั้นตอนการขอลบบัญชี

  - คลิกปุ่ม "ลบบัญชี" เพื่อเปิดหน้าต่างป๊อปอัพ "ขอลบข้อมูลส่วนบุคคล"
  [หน้าต่างขอลบข้อมูล](./images/ui-user-request-popup.png)

  - ระบุสาเหตุ: เลือกเหตุผลจากเมนู Dropdown เช่น กังวลเรื่องความเป็นส่วนตัว หรือไม่ต้องการใช้งานต่อ
  [รายการเหตุผล](./images/ui-user-reason-list.png)

  - รายละเอียดเพิ่มเติม: กรอกข้อมูลเพิ่มเติมในช่องที่กำหนด (ถ้ามี)

  - ยืนยันตัวตน: พิมพ์ข้อความว่า "ยืนยันการลบ" เพื่อยืนยันความต้องการ

  - กดปุ่ม "ยืนยันการลบข้อมูล" เพื่อส่งคำขอ
  [ยืนยันคำขอ](./images/ui-user-confirm-input.png)

1.3 ผลกระทบหลังส่งคำขอ

  - คำขอจะถูกส่งไปยังระบบ Admin เพื่อตรวจสอบ

  - บัญชีจะถูกปิดการใช้งานทันที และข้อมูลจะถูกลบภายใน 30 วัน

2. สำหรับผู้ดูแลระบบ (Admin)
2.1 ดูรายการคำขอลบบัญชี

  - ไปที่เมนู "All Request" เพื่อดูรายการคำขอทั้งหมด ซึ่งจะแสดงชื่อผู้ใช้, ประเภทคำขอ และสถานะ
  [รายการคำขอทั้งหมด](./images/ui-admin-all-requests.png)

2.2 ดูรายละเอียดคำขอ

  - คลิกไอคอน "ดวงตา" เพื่อดูรายละเอียดเหตุผลที่ผู้ใช้ระบุมา
  [รายละเอียดคำขอ](./images/ui-admin-request-detail.png)

2.3 อนุมัติ/ปฏิเสธคำขอ

  - อนุมัติ: กดปุ่มสีเขียวเพื่อดำเนินการลบข้อมูล

  - ปฏิเสธ: กดปุ่มสีแดงหากคำขอไม่ถูกต้อง

2.4 ตรวจสอบประวัติการดำเนินการ (Audit Log)
[ประวัติระบบ Audit Log](./images/ui-admin-audit-log.png)

แอดมินสามารถตรวจสอบประวัติการทำรายการย้อนหลังทั้งหมดในระบบ เพื่อความโปร่งใสและตรวจสอบได้ตามมาตรฐาน PDPA:

  - การเข้าถึง: คลิกที่เมนู "Audit Log" จากแถบเมนูด้านซ้ายมือ

  - รายละเอียดที่แสดง: ระบบจะแสดงตารางประวัติ ซึ่งประกอบด้วย:

  - สร้างเมื่อ: วันที่และเวลาที่เกิดรายการ

  - เจ้าของคำร้อง: ชื่อและอีเมลของผู้ใช้งานที่เกี่ยวข้อง

  - คำร้อง: ประเภทของรายการ (เช่น ขอลบบัญชี, แจ้งเหตุการณ์)

  - การดำเนินการ: สถานะการจัดการ (เช่น อนุมัติ, ปฏิเสธ, สร้างคำร้อง)

  - ดำเนินการโดย: ชื่อของผู้ที่ทำรายการนั้นๆ (แอดมิน หรือ ผู้โดยสาร)
 
  - การค้นหาและคัดกรอง: สามารถเลือกดูตามประเภท Action, ประเภทคำร้อง หรือเลือกช่วงวันที่ต้องการตรวจสอบได้จากแถบด้านบน

3. สถานะคำขอลบบัญชี
  - รอการดำเนินการ (สีเหลือง): คำขอใหม่ที่รอตรวจสอบ

  - อนุมัติแล้ว (สีเขียว): ดำเนินการลบข้อมูลสำเร็จ

  - ปฏิเสธแล้ว (สีแดง): คำขอไม่ได้รับการอนุมัติ

4. เหตุผลในการขอลบบัญชี
  - ผู้ใช้สามารถเลือกเหตุผลได้ดังนี้:

  - ไม่ต้องการใช้บริการอีกต่อไป 

  -  กังวลเรื่องความเป็นส่วนตัว

  - ข้อมูลไม่ถูกต้อง/ไม่เป็นปัจจุบัน

  - ไม่พึงพอใจกับบริการ

  - ข้อมูลรั่วไหลหรือมีปัญหาด้านความปลอดภัย

5. คำชี้แจงการใช้ AI
ทีมพัฒนาได้นำ ChatGPT มาใช้เป็นเครื่องมือสนับสนุนใน ระดับ 2 ดังนี้:

ใช้ในขั้นตอนดังนี้
- วิเคราะห์และออกแบบ Database Schema (Prisma)
- ออกแบบ Request System และ Ticket System
- ออกแบบ Audit Log ให้สอดคล้องกับ PDPA
- เสนอแนวทาง implementation ให้สอดคล้องกับกฎหมาย
- ตรวจสอบ Prisma schema
- วิเคราะห์ Vue UI binding error
- แนะนำการแก้ไข bug และโครงสร้างโค้ด
- แนะนำโครงสร้างข้อมูลสำหรับแสดงผลหน้า UI
- อธิบายแนวคิด Soft Delete
- แนะนำแนวทาง Audit Logging
- อธิบาย Cron Job และ Data Retention Policy

ใช้เพื่ออะไรบ้างในเเต่ละขั้นตอน
- ช่วยอธิบายแนวคิดทางเทคนิค
- ช่วยวิเคราะห์และออกแบบโครงสร้างระบบ
- ช่วยตรวจสอบความถูกต้องของ schema และ logic
- ช่วยให้เข้าใจข้อกำหนดทางกฎหมาย (PDPA)
